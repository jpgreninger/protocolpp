jikencrypt Class Reference

#include "include/jikencrypt.h"

Detailed Description

IKE Encrypt and Integrity Functions

Class to provide all the encryption and authentication algorithms and functionality needed for Internet Key Exchange (IKEv2). Also provides an interface to the Linux kernel to retrieve data from the hardware random number generator. See \kcapi for additional details about the kernel interface

The class has a single constuctor that requires a logger object and randomizer as shown below

jikencrypt(std::shared_ptr<InterfacePP::jlogger>& logger,
std::shared_ptr<jrand>& rand);
jikencrypt

Accessor function to provide direct access to the Kernel's hardware random number generator. Requires a byte pointer and the length required. If not available due to lack of support in the kernel, returns data from the jrand object call to crypto() to retrieve data from RANDP. Returns 0 on success, non-zero otherwise

int get_random(uint8_t *buf,
uint32_t len);

Encryption method for IKEv2. Accepts the Security Association created during the initial 4-way handshake. Input can be any amount of data, with plaintext or ciphertext returned in the output pointer. If the cipher uses an IV, it can be passed as the last function

bool encrypt(direction_t dir,
std::shared_ptr<jikev2sa>& sa,
jarray<uint8_t>& input,
std::shared_ptr<jarray<uint8_t>>& output,
std::shared_ptr<jarray<uint8_t>>& iv);
jarray

Authentication method for IKEv2. Used when the cipher chosen by the responder requires a separate authenticator (AES-CBC, AES-CTR). Not needed for AEAD ciphers such as AES-GCM, AES-CCM, CHACHA20-POLY1305

bool integrity(direction_t dir,
std::shared_ptr<jikev2sa>& sa,
jarray<uint8_t>& input,
std::shared_ptr<jarray<uint8_t>>& icv);
