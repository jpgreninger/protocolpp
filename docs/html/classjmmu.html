Detailed Description

Management Unit for Protocol++® (ProtocolPP®) testbench

Memory management class for Protocol++® (ProtocolPP®) testbench to track and manage all memory allocation done during testbench operation. This was necessary to remove any memory leaks from taking all resources from the system

The default constructor creates and empty memory manager as shown below

jmmu mmu;

To start tracking a packet with it's input, output, and expected pointers a user would call the track function with the name of the packet and it's associated pointer in a vector as below

jarray<uint8_t> pktin(20,0);
jarray<uint8_t> pktout(20,0);
jarray<uint8_t> pktexp(20,0);
std::vector<uint8_t*> pkt1_pointers = {pktin.get_ptr(),
pktout.get_ptr(),
pktexp.get_ptr()};
mmu.track(std::string("packet1", pkt1_pointers));

You could also initialize the pointer vector to empty and use an iterator to push the pointers into the vector as they become available

The two set_mem() methods allow either packet pointers or stream/flow pointers to be added to the MMU as follows

mmu.set_mem(std::string("packet2", pkt2in.get_ptr());

Internally the set_mem() method looks up the packet and adds the pointer to it's vector of pointers

The other set_mem() is meant to track security associations and the memory associated with them as follows

std::shared_ptr<jipsecsa> flow1 = std::make_shared<jipsecsa>();
mmu.set_mem(std::string("flow1", flow1.get_ptr()));

The free functions free either a single packet or flow or frees all remaining packets and flows from the MMU as follows

// free memory of a single packet
mmu.free(std::string("packet1"));
// free all remaining packets and flows
mmu.free();
